# Dashboard: global and customizable view

The **Dashboard** provides an overall view, or a targeted view through filters, of all records linked to counterparties*.

## Display customization

Each user can **customize the table display** by selecting the columns they want to see.

## Column details

| **Column** | **Description** |
|--------------|----------------|
| **Name** | Displays the name of the file, client, or entity. |
| **Type** | Indicates the counterparty type (e.g., Individual, Legal Entity, Fund). |
| **Completion** | Shows the progress level of the file. |
| **Status** | Indicates the current file status (e.g., in progress, approved, rejected, waiting for information). |
| **Group name** | Refers to the Group (firm) or organization the file belongs to. |
| **Subscription** | Indicates whether a subscription is currently in progress for the file. |
| **Risk** | Shows the assigned risk level after assessment (e.g., low, medium, high). |
| **Next periodic review** | Planned date for the next verification or update of the file. |
| **Days before next periodic review** | Number of days remaining before the planned review date. |
| **Last action** | Summarizes the latest action performed on the file (e.g., document update, status change). |
| **Owner** | Person who created the file or was assigned responsibility for follow-up. |
| **Assigned to** | Name of the user or analyst currently in charge of the file. |
| **Secure link expiration** | Expiration date of the link sent to the counterparty to complete information. |
| **Files under review** | Indicates the periodic review date has been reached or an attached document has expired. |
| **New match** | Indicates a newly detected match in screening databases (e.g., sanctions, PEP, adverse media). |

---

## Filters and export

- **Filters** are available for each table column, and at the top of the page (e.g., by **fund**, **groups**, or **statuses**).  
- Data **export** respects applied filters: only records visible on screen are included in the exported file.

---

{% admonition type="info" %}
**Note:**  
A *counterparty* corresponds to a subsection in **Diligence**.  
Available counterparty types are: **investor**, **service provider**, **asset**, **general partner**, and **distributor**.
{% /admonition %}
