# Dashboard : vue globale et personnalisable

Le **Dashboard** permet d’avoir une vision d’ensemble — ou ciblée grâce aux filtres — de l’ensemble des fiches liées aux contreparties*.

## Personnalisation de l’affichage

Chaque utilisateur peut **personnaliser l’affichage du tableau** en sélectionnant les colonnes qu’il souhaite visualiser.

## Détails des colonnes

| **Colonne** | **Description** |
|--------------|----------------|
| **Nom** | Affiche le nom du dossier, du client ou de l’entité. |
| **Type** | Indique le type de la contrepartie (ex. : Personne Physique, Personne Morale, Fonds). |
| **Complétude** | Montre le niveau d’avancement du dossier. |
| **Statut** | Précise l’état actuel du dossier (ex. : en cours, validé, rejeté, en attente d’informations). |
| **Nom du groupe** | Fait référence au Groupe (cabinet) ou à l’organisation auquel le dossier appartient. |
| **Souscription** | Permet de savoir si une souscription est actuellement en cours pour le dossier. |
| **Risque** | Indique le niveau de risque attribué après l’évaluation (ex. : faible, moyen, élevé). |
| **Prochaine revue périodique** | Date prévue pour la prochaine vérification ou mise à jour du dossier. |
| **Jours avant la prochaine revue périodique** | Nombre de jours restants avant la date de revue planifiée. |
| **Dernière action** | Résume la dernière opération effectuée sur le dossier (ex. : mise à jour des documents, changement de statut). |
| **Propriétaire** | Personne ayant créé le dossier ou désignée comme responsable de son suivi. |
| **Assigné à** | Nom de l’utilisateur ou de l’analyste actuellement chargé du dossier. |
| **Expiration du lien sécurisé** | Date limite de validité du lien envoyé à la contrepartie pour compléter des informations. |
| **Dossiers en revue** | Indique que la date de revue périodique est atteinte ou qu’un document associé est expiré. |
| **Nouveau match** | Signale la détection d’une nouvelle correspondance dans les bases de données de screening (ex. : sanctions, PEP, adverse media). |

---

## Filtres et export

- Des **filtres** sont disponibles sur chaque colonne du tableau, ainsi qu’en haut de la page (ex. : par **fonds**, **groupes**, ou **statuts**).  
- L’**export** des données tient compte des filtres appliqués : seules les fiches visibles à l’écran seront incluses dans le fichier exporté.

---

{% admonition type="info" %}
**Note :**  
Une *contrepartie* correspond à une sous-section dans **Diligence**.  
Les types de contreparties disponibles sont : **investisseur**, **prestataire**, **asset**, **general partner** et **distributeur**.
{% /admonition %}
